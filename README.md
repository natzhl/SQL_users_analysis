# SQL_users_analysis
Данные получены из public dataset Google BigQuery thelook_ecommerce:  https://console.cloud.google.com/marketplace/product/bigquery-public-data/thelook-ecommerce?hl=en&project=causal-flame-283208

**Цель проекта** изучить набор данных с помощью sql-запросов, проанализировать пользовательскую активность, расчитать ключевые продуктовые метрики (MAU, DAU, Retention, LTV) и выявить точки роста для повышения эффективности бизнеса, улучшения конверсии и увеличения выручки.

Это B2C модель e-commerce магазин одежды, поэтому **ключевые KPI**:

1. Пользователи и retention

- Активные пользователи (DAU / MAU) — по events
- Повторные покупки / Retention Rate = % клиентов, вернувшихся за второй заказ
- LTV (Lifetime Value) = доход с одного клиента за весь период активности
- Портрет пользователя (пол, возраст, страна)


2. Продажи и доход

- Выручка (Revenue) = сумма всех заказов за период
- Средний чек (AOV) = Revenue / Количество заказов
- Количество заказов / конверсия (Conversion Rate) = % пользователей, сделавших заказ
- Выручка по категориям и продуктам → определить “хиты” и товары с низкой оборачиваемостью

3. Маржинальность и unit-экономика

- Маржинальность = выручка - себестоимость (~60 % от цены) 
- Gross Margin на товар / заказ = Revenue − COGS
- Unit-экономика (экономика одного заказа, экономика на пользователя)

4. Маркетинговые и поведенческие метрики

- Конверсия событий → заказ (events → orders)
- Пути пользователя: какие страницы и события приводят к покупке

# Выводы

``` Бизнес приносит стабильную прибыль с каждого заказа, но рост дохода можно усилить через повышение конверсии на этапе главной страницы и развитие низкооборотных товаров. ```

Ядро аудитории (пользователи заходящие ежедневно всего 5%)

→ Развивать каналы вовлечения для регулярных посещений → увеличить sticky factor.
Персонализировать рекомендации для новых пользователей, чтобы повышать LTV с первых заказов.
Сегментировать пользователей по стране и возрасту для таргетированной коммуникации.

Общая выручка: 2 699 486, средний чек: 86.09.
Среднее количество товаров в заказе: 1.89, то есть большинство пользователей покупают 1–2 товара.
ТОП-5 продуктов формируют значительную часть дохода.

→ Фокусироваться на продвижении хитов → увеличить продажи наиболее популярных товаров.
Анализировать низкооборотные товары: возможно, стоит снизить их закупки или предлагать акции.
Увеличивать средний чек через кросс-продажи и подбор комплектов (bundle).

Основная потеря пользователей на этапе Home → Product (11%).
Почти все пользователи, добавившие товар в корзину, доходят до покупки → высокая эффективность корзины.

→ Оптимизировать главную страницу → стимулировать переход на карточки товаров (баннеры, рекомендации, CTA).
Тестировать элементы на главной странице: подборки, «топ товаров», персонализированные рекомендации.
Использовать A/B тесты, чтобы понять, какие элементы увеличивают переход на просмотр товара.

→ Отказаться от убыточных позиций.

Рассчитать unit-экономику по категориям, чтобы видеть, какие товары приносят максимальную прибыль на одного пользователя.
